Voici un résumé rapide de la réunion concernant le processus MyAccess, expliqué en français :

Le Nouveau Processus MyAccess

Ce que c'est : MyAccess est la nouvelle méthode pour demander l'accès à la plateforme CIDP. L'ancien fichier Excel ne doit plus être utilisé pour effectuer ces demandes.
+1


Le flux d'approbation : Lorsqu'un accès est demandé, il doit d'abord être validé par le manager direct de l'utilisateur. Ensuite, il passe à un deuxième niveau de validation par des managers supérieurs, comme les responsables des ventes ou des réclamations.
+1


Mise en œuvre : Une fois la demande approuvée dans le système, c'est le groupe de services (spécifiquement Julie Sanchez) qui est responsable d'attribuer concrètement les accès, et non l'équipe SESAM.
+1

Le Problème Principal : Les Managers en Congé

Le problème : L'équipe a rencontré un blocage de processus à cause d'une manager (Jenny) partie en congé maternité, ce qui a laissé son équipe sans personne pour approuver les demandes d'accès.
+3


La fonctionnalité de délégation : Il existe une option dans MyAccess permettant aux managers de déléguer leurs approbations à une autre personne pendant leur absence. Le problème survient si le manager oublie de configurer cette délégation avant son départ.
+3


L'action à suivre : Caroline va se renseigner auprès de l'équipe du siège (HO) pour savoir comment procéder ou modifier le flux si un manager est absent et que la délégation n'a pas été configurée.
+2

Suivi des Données

Problème technique : L'extraction des données pour voir les utilisateurs actifs échoue actuellement dans ServiceNow.


Solution temporaire : En attendant de résoudre ce problème de système, l'équipe peut partager l'ancien fichier Excel en tant que référence temporaire, bien qu'il puisse manquer les demandes les plus récentes.
